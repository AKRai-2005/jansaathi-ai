## SLIDE 4: PROCESS FLOW DIAGRAM

**Title:** How JanSaathi Works — End-to-End Flow

**Content — create a FLOWCHART diagram with these steps (use colored boxes and arrows):**

START → User sends message on Telegram

BRANCH: Is it text or voice?

TEXT PATH (blue boxes):
1. AWS Translate — Detect Language
2. AWS Translate — Convert to English
3. AWS Comprehend — Analyze Sentiment
4. AWS Bedrock (Claude 3 Haiku) — Extract User Profile (age, occupation, state, income, caste, BPL, etc.)
5. DynamoDB — Match & Rank Schemes (relevance scoring algorithm)
6. AWS Bedrock — Generate Personalized Response  
7. AWS Translate — Translate back to user's language
8. Send text response + scheme cards on Telegram

VOICE PATH (green boxes):
1. Download audio from Telegram
2. Groq Whisper — Instant Transcription (primary)
3. If fails → AWS S3 → AWS Transcribe (fallback)
4. → Merges into Text Path at step 3 (Comprehend)

OPTIONAL BRANCH (orange):
5b. AWS Polly — Generate voice reply
5c. Send voice note on Telegram

END → User receives matched schemes in their language

- Citizen: Send text query, Send voice query, View schemes, Receive alerts
- Admin: Seed scheme data, Monitor health
